# TIXANO CODE BASE FOLDER STRUCTURE

## Overview
Tixano is a Next.js web application (MeshJS starter template) integrated with Cardano blockchain for event ticketing NFTs. The project combines a Next.js frontend with Aiken smart contracts, email integration, IPFS storage, and Supabase backend.

---

```
tixano/
│
├── Configuration & Build Files
├── Source Code (/src)
├── Smart Contracts (/contracts)
```

```
│                                          Contains: NEXT_PUBLIC_*, API keys, Supabase URLs, Gmail credentials, IPFS keys

├── .git/                               ← Git repository metadata (version control)
```

## Key Project Features

### Architecture Layers
- **Frontend**: React + Next.js (TypeScript)
- **Backend**: Next.js API routes
- **Blockchain**: Cardano + Aiken smart contracts
- **Database**: Supabase (PostgreSQL)
- **File Storage**: IPFS (via Pinata) + Supabase Storage
- **Email**: Gmail API
- **Styling**: Tailwind CSS

### Main Workflows
1. **Event Creation**: User creates event → API stores in Supabase
2. **Ticket Minting**: Admin initiates → Smart contract mints NFT → Image uploaded to IPFS
3. **Ticket Claim**: User claims ticket → Confirmation email sent via Gmail
4. **Wallet Integration**: MeshJS connects to Cardano wallets

## Next Steps for Development

1. **First-time setup**: Copy `.env.example` → `.env.local` and fill in values
2. **Install dependencies**: `npm install`
3. **Build contracts**: `cd contracts && aiken build`
4. **Start dev server**: `npm run dev`
5. **Review setup guides**: See `setup/ENV_SETUP.md` and `mail/GMAIL_SETUP.md`
